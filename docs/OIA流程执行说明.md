# OIA (One Intent, All Agents) 多智能体协同执行标准
## 🎯 核心目标
一个用户意图，所有专业Agent自动协同完成，最大化专业能力，最小化人为干预。

---
## 🔒 强制执行规则
### 1. 会话初始化强制流程
✅ **必须**在响应用户前执行：
```bash
node ~/.openclaw/workspace/skills/main-agent/init-script.js
```
❌ 初始化失败不得响应用户任何请求

### 2. 意图处理强制流程
```
用户请求 → 意图解析 → 全局感知 → 任务分发 → 并行执行 → 交叉验证 → 结果汇总 → 质量审核 → 交付用户
```
❌ 不得跳过任何环节
❌ 不得由主Agent直接处理专业领域任务

### 3. 任务分发规则
| 任务类型 | 必须分配给 | 必须输出 |
|----------|------------|----------|
| 项目类（迭代/规划/发布） | PMAgent | 项目计划、进度表、质量标准 |
| 文档类（规范/报告/教程） | DocAgent | 符合格式规范的文档、校验报告 |
| 代码类（开发/工具/脚本） | CodeAgent | 可运行代码、注释、测试用例 |
| 测试类（验证/质量/兼容） | TestAgent | 测试报告、通过率报告、问题清单 |
| 发布类（部署/上线/打包） | OpsAgent | 发布清单、版本标签、上线报告 |
| 风险类（合规/评审/安全） | RiskAgent | 风险评估报告、合规检查报告 |

### 4. 交叉验证规则
✅ 所有输出必须经过至少2个不同Agent的交叉验证
✅ 代码必须经过TestAgent测试
✅ 文档必须经过RiskAgent合规检查
✅ 发布必须经过PMAgent版本验证

### 5. 异常处理规则
- 单个Agent执行失败自动重试3次
- 重试失败自动升级任务复杂度，分配更高优先级Agent
- 连续失败自动通知用户，给出替代方案
- 所有异常自动记录到日志，用于后续优化

---
## ✅ 验证标准
### 正确执行特征
- 响应中明确说明各Agent的分工和输出
- 包含各Agent的执行过程和结果
- 有明确的交叉验证报告
- 最终结果符合质量标准

### 错误执行特征
- 主Agent直接给出结果，没有提到专业Agent的参与
- 没有过程记录，只有最终输出
- 缺少交叉验证环节
- 结果不符合专业规范
