# Agent 任务追踪系统 - 使用指南

## 🎯 系统概述

这是一个完整的 Agent 任务追踪和监控系统，用于实时监控所有 Agent 的执行情况、活跃状态、完成进度，并提供详细的进展记录。

## 📁 目录结构

```
agent-tracker/
├── SKILL.md              # 技能说明文档
├── tracker.py            # 追踪脚本
├── README.md            # 本文件
├── state/               # 状态存储
│   ├── agents.json      # Agent 状态
│   ├── sessions.json    # 会话状态
│   └── current-tasks.json # 任务状态
├── logs/                # 日志目录
│   └── YYYY-MM-DD/     # 按日期组织的日志
│       ├── agent-activity.jsonl
│       └── interventions.json
└── reports/             # 报告目录
    └── daily-summary-YYYY-MM-DD.md
```

## 🚀 快速开始

### 1. 完整状态扫描

要进行一次完整的状态扫描，我会：

1. 运行 `agents_list` - 获取所有可用 Agent
2. 运行 `sessions_list` - 获取所有活跃会话
3. 运行 `subagents` - 获取子 Agent 状态
4. 保存所有状态到 JSON 文件
5. 生成每日摘要报告

### 2. 定期检查

**建议频率：每 15-30 分钟一次**

在日常对话中，我会定期检查状态并记录变化。

## 🔧 核心功能

### 实时监控

- **Agent 列表** - 所有可用 Agent 的状态
- **活跃会话** - 正在进行的对话和任务
- **子 Agent** - 监控子 Agent 的运行状态
- **Token 使用** - 实时统计资源消耗

### 任务追踪

- 任务开始/结束时间
- 执行进度和里程碑
- 成功/失败状态
- 输出结果摘要

### 干预支持

**自动检测需要干预的情况：**

- ⚠️ 会话异常终止
- ⚠️ 任务运行超时
- ⚠️ Token 使用异常
- ⚠️ Agent 无响应

## 📊 报告说明

### 每日摘要报告

位置：`reports/daily-summary-YYYY-MM-DD.md`

包含内容：
- Agent 状态列表
- 活跃会话详情
- Token 使用统计
- 需要关注的事项

### 活动日志

位置：`logs/YYYY-MM-DD/agent-activity.jsonl`

使用 JSON Lines 格式记录所有活动，便于分析和查询。

## 💡 使用示例

### 查看当前状态

只需要说：
- "查看当前 Agent 状态"
- "现在有哪些活跃会话？"
- "生成今日追踪报告"

### 定期检查

我会在日常对话中自动进行定期检查，你也可以随时要求：
- "进行一次完整的状态扫描"
- "检查是否有需要干预的情况"
- "更新追踪记录"

## 🔍 监控指标

### 关键指标

1. **Agent 活跃度** - 在线/离线状态
2. **会话健康度** - 是否正常运行
3. **任务完成率** - 成功/失败比例
4. **资源使用量** - Token 消耗统计
5. **响应时间** - 执行效率监控

### 异常检测

- 会话异常终止 (`abortedLastRun: true`)
- Token 突然大幅增加
- 长时间无更新的会话
- 失败的任务执行

## 📝 数据记录

### 状态快照

每次扫描都会保存完整的状态快照，便于：
- 历史对比分析
- 问题回溯调试
- 趋势分析预测

### 详细日志

所有活动都有时间戳记录，包括：
- 状态变化时间
- 操作类型
- 详细数据
- 关联的 Agent/会话

---

**记住：** 这个追踪系统的目标是让你能够全面、及时地了解所有 Agent 的状态，以便进行有效的管理和及时的干预！
