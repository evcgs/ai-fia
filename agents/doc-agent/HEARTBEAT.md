# HEARTBEAT.md - Agent 追踪系统

## 📋 定期检查任务

**每 15-30 分钟执行一次：**

### 1. Agent 状态检查
- 运行 `agents_list` 获取可用 Agent 列表
- 记录 Agent 状态变化
- 检查是否有新 Agent 加入或离线

### 2. 会话状态检查
- 运行 `sessions_list` 获取活跃会话
- 检查会话是否异常终止 (`abortedLastRun`)
- 监控 Token 使用情况
- 记录会话更新时间

### 3. 子 Agent 检查
- 运行 `subagents` 检查子 Agent 运行状态
- 监控长时间运行的任务
- 检测是否有需要终止的异常子 Agent

### 4. 状态保存和报告
- 更新 `agent-tracker/state/` 中的状态文件
- 记录活动日志到 `agent-tracker/logs/`
- 生成每日摘要报告（如需要）

### 5. 干预检查
- 检测异常终止的会话
- 检查超时任务
- 识别异常的资源使用
- 如果发现需要干预的情况，立即通知用户

---

**重要：** 保持检查频率适中，避免过度消耗资源。只有在发现需要干预的情况时才主动联系用户。
